# Set up STATE Discussion category (Boss action)

## Steps

1. Go to https://github.com/ruijayfeng/aisurface/settings (admin only)
2. Enable Discussions (if not already): "Set up discussions" → choose "Announcements" preset (we'll customize)
3. Create categories:
   - **State of aisurface** (Announcement type) — for monthly digests
   - **Showcase** (Discussion type) — for users sharing their results
   - **Help** (Q&A type) — for usage questions
   - **Ideas** (Discussion type) — for feature requests
4. Pin a "Welcome" discussion in Showcase
5. Add a Discussion link to README.md footer
6. Update this file with the URLs once created

## Why these 4 categories

- **State of aisurface** (Announcement) — closed thread, only maintainers post. The monthly digest lands here so subscribers get one canonical post per month.
- **Showcase** (Discussion) — open, users share their before/after. Feeds back into case-studies/.
- **Help** (Q&A) — structured Q&A. GitHub marks accepted answers; this becomes a knowledge base.
- **Ideas** (Discussion) — feature requests with upvote signal. Helps the monthly digest's "roadmap" section.
